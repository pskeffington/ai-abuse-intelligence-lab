# Ethics

This project is intended for defensive research, platform integrity analysis, and reproducible analyst workflows.

## Principles

- Use authorized, synthetic, or properly governed data.
- Minimize personal data wherever possible.
- Preserve provenance without exposing private source material.
- Treat outputs as review signals rather than final claims.
- Document uncertainty and alternative explanations.
- Avoid stigmatizing individuals or groups based on weak signals.

## Responsible use

Do not use this project to automate abuse, impersonation, harassment, surveillance of private persons, evasion, credential misuse, or unauthorized access.

## Reporting language

Recommended language:

- review signal
- repeated indicator
- timing concentration
- graph relationship
- requires corroboration

Avoid unsupported language:

- proves coordination
- identifies perpetrator
- confirms intent
- attributes campaign

## Data minimization

Use redacted labels, hashes, or synthetic values when full raw content is not required for reproducibility.
